# W25Q64 参数存储布局

## 分区

| 地址范围 | 大小 | 用途 |
|---|---:|---|
| 0x000000–0x000FFF | 4 KiB | 旧版原始 `PARAM_T` 只读迁移源；新固件不擦写 |
| 0x001000–0x001FFF | 4 KiB | 参数槽 A |
| 0x002000–0x002FFF | 4 KiB | 参数槽 B |
| 0x003000–0x7FFFFF | 其余 | 未分配，当前固件不访问 |

## 版本 2 记录

记录是固定小端格式，不依赖运行时 `PARAM_T` 的编译器填充。

| 偏移 | 长度 | 字段 |
|---:|---:|---|
| 0 | 4 | magic `CHPM`（0x4D504843） |
| 4 | 2 | schema version = 2 |
| 6 | 2 | header size = 16 |
| 8 | 4 | 单调 sequence，按有符号差比较处理回绕 |
| 12 | 2 | payload size = 10 |
| 14 | 2 | reserved = 0 |
| 16 | 1 | Modbus 地址 1..247 |
| 17 | 1 | 风扇模式 0/1 |
| 18 | 2 | 手动 PWM 4000..10000 |
| 20 | 2 | 自动 PWM 4000..10000 |
| 22 | 4 | 重启计数 |
| 26 | 4 | IEEE CRC32，覆盖偏移 0..25 |
| 30 | 4 | commit = 0xA55AC33C |

## 原子提交与恢复

保存只在配置实际变化时发生。固件选择非活动槽，擦除整个 4 KiB 扇区，先写入 commit 为 0xFFFFFFFF 的记录并逐字节回读校验，再单独编程 commit 字段，最后重新读取并执行完整格式、范围和 CRC 验证。只有全部成功后才切换活动槽；失败时恢复运行时参数为最近一次已持久化值。

启动时独立校验两个槽并选择较新的有效 sequence。最新槽损坏、写到一半或 commit 缺失时自动回退到旧槽。两个槽都无效时读取地址 0 的旧版 16 字节结构（版本 0x00000101）；范围合法则迁移，否则使用默认值并创建首条版本 2 记录。

## 可靠性边界

- SPI DMA 传输、页编程、扇区擦除和整片擦除均有超时，不再无限等待 BUSY。
- 写入跨页自动切分；每次提交有回读验证。
- 两槽交替使任意一次掉电至少保留前一有效配置，并把擦除分散到两个扇区。
- 参数只在地址、模式或配置 PWM 变化时保存；传感器和实时指标不写 Flash，避免高频磨损。
- 尚未用真实 W25Q64 做断电注入、最大擦除时间和寿命抽测；这些列入实机验证。
