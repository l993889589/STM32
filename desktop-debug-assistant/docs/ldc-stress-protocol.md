# LDC Stress Protocol

This protocol is used by LeduO Debug Assistant to stress-test MCU LDC receive paths over USB CDC.

## Request Frame

All multi-byte fields are big-endian except CRC16, which uses Modbus RTU little-endian order.

```text
AA 55
LEN_H LEN_L
SEQ_3 SEQ_2 SEQ_1 SEQ_0
TYPE
PAYLOAD...
CRC_L CRC_H
```

Fields:

- Header: `AA 55`
- LEN: body length, from `SEQ` through the end of `PAYLOAD`
- SEQ: 32-bit incrementing sequence number
- TYPE: currently `0x01` for echo/ack test
- PAYLOAD: generated by the assistant
- CRC16: Modbus CRC16 over `AA 55 + LEN + BODY`

## ACK Frame

```text
55 AA
00 05
SEQ_3 SEQ_2 SEQ_1 SEQ_0
STATUS
CRC_L CRC_H
```

Fields:

- Header: `55 AA`
- LEN: fixed to `5`
- SEQ: echoed request sequence number
- STATUS: `0x00` means OK; non-zero means MCU-side error
- CRC16: Modbus CRC16 over `55 AA + LEN + BODY`

## Test Modes

The assistant can split one request frame into smaller serial writes:

- Chunk = `0`: write the whole frame once
- Chunk > `0`: write frame in chunks of that byte count
- Chunk gap: delay between chunks, in milliseconds

This is intended to test LDC behavior under complete-frame writes, partial-frame writes, and sticky multi-frame serial streams.
