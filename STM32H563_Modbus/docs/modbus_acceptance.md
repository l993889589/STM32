# Modbus 实机验收记录

日期：2026-07-11

## 测试链路

```text
desktop-debug-assistant
  -> COM3 / USB-RS485
  -> 板载 RS485-1 / MAX13487
  -> USART2 PA2/PA3, 115200 8N1
  -> LDC 静默间隔分帧
  -> ld_modbus RTU 从站（站号 1）
```

固件由 H7-TOOL 通过 CMSIS-DAP/pyOCD 执行整片擦除、烧录、校验和复位，
镜像及向量表起始地址均为 `0x08000000`。

## 已通过项目

1. FC03 读取身份保持寄存器。
2. FC04 读取输入寄存器。
3. FC06 写单个保持寄存器。
4. FC10 写多个保持寄存器。
5. FC16 掩码写寄存器。
6. FC17 写/读多个寄存器。
7. FC05 + FC01 单线圈写读。
8. FC0F + FC01 多线圈写读。
9. FC02 读取离散输入。
10. 非法地址返回异常码 02。
11. 非法功能返回异常码 01。
12. CRC 错误请求保持静默。
13. 错误站号请求保持静默。
14. 广播写不响应。
15. 广播写实际生效。
16. 故障注入后可继续完成正常请求。

结果：16/16 通过。

## 仍需区分的边界

- 当前验收证明 standalone 固件和 RS485-1 链路可工作，不证明原 Boot 的
  签名、OTA 或回滚链路。
- TCP 主从目前在桌面端以真实 socket 完成主从联调；H563 本工程尚未接入
  以太网或网络控制器，因此板端 TCP 仅验证了协议核心的主机测试。
- RTU 主机协议接口已有主机单元测试；当前板端示例应用运行的是从机角色。
