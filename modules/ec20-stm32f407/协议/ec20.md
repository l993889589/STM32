## 一，串口AT指令



### 1，AT+LOG

 功能：打开和关闭LOG信息

 

```
发送:
	:AT+LOG=“1”\r\n  打开LOG信息
返回：
	+LOG="1"
	OK
	
发送:
	:AT+LOG=“0”\r\n  关闭LOG信息
返回：
	+LOG="0"
	OK	
```



### 1，ATI 

 功能：查询软件版本号

 查询:ATI\r\n

```
发送:
	ATI\r\n
返回：
	+LOG="1"
	OK
```

### 7，AT+RELAY

功能：继电器操作和查询

查询：AT+RELAY?\r\n

```
发送：AT+RELAY?\r\n
返回：
	relay number:4          设备一共有四个继电器
	relay state:1,0,0,0     继电器当前状态： 继电器1打开，2关闭，3关闭，4关闭
```

- 操作单个继电器：AT+RELAY=%d,%d   

```
发送:AT+RELAY=1,1  打开继电器1
返回:relay_1打开

发送:AT+RELAY=1,0  关闭继电器1
返回:relay_1关闭
```

- 批量操作继电器:AT+RELAYALL=%d,%d,%d,%d    依次输入继电器的状态

```
发送:AT+RELAYALL=1,0,1,0\r\n  继电器1打开，2关闭，3打开，4关闭
返回:
	relay_1打开
	relay_2关闭
	relay_3打开
	relay_4关闭
	relay state:1,0,1,0
```



### 8，AT+INPUT?

功能：查询输入状态

查询:AT+INPUT?\r\n   

```
发送:AT+INPUT?\r\n  
返回:
	input number:4         一共有四路输入
	input state:1,1,1,1    输入低电平有效，所以这四路的状态都是无输入
```

## 二，网络协议

```

设备支持TCP/UDP/FTP/HTTP/MQTT

版本号：v1.0 时间2023/5/14
```

现有设备采集的信息包括：

四路输入：relay1,relay2,relay3,relay4

四路输入：input1,input2,input3,input4

温湿度：temp,humi

一路电压：bsp未完成，假数据

5路电流：bsp未完成，假数据



### 1，数据格式



设备所有报文均为JSON格式，支持时间戳，支持协议定制

```
不带时间戳数据格式：

{
    "humi": "44",
    "relay1": "0",
    "relay2": "0",
    "relay3": "0",
    "relay4": "0",
    "temp": "30"
}
```

```
带时间戳数据格式：

{
    "humi": "44",
    "relay1": "0",
    "relay2": "0",
    "relay3": "0",
    "relay4": "0",
    "temp": "30",
     "time": "2023-05-12"
}
```



###  2，数据上报

- 主动上报
